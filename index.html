XPath locator is incorrect Verify the xpath locator using the Google Chrome console. 

      $x("//div[@id='content']")

@FindBy locator type is mismatched The example shows an id locator type incorrectly supplied with an xpath. 

      @FindBy(id = "//h1[text()='Page Header']")
      WebElement pageHeader;

Element is within an iframe Switch to the iframe to identify the element. 

      // By index
      driver.switchTo().frame(0);

      // By name or id
      driver.switchTo().frame(frameName);

      // By WebElement
      @FindBy(xpath = "//iframe")
      private WebElement iframe;

      driver.switchTo().frame(iframe);

      // Switch to main content
      driver.switchTo().defaultContent();

Driver must wait for element Use Serenity or Selenium explicit waits to wait for expected conditions. Serenity is the preferred method. 

      // Serenity

      // Wait for element to be clickable with a 10 second timeout
      @FindBy(xpath = "//button[text()='Next']")
      WebElementFacade nextButton;

      nextButton.withTimeoutOf(Duration.ofSeconds(10)).waitUntilClickable().click();

      // Chain expected conditions to wait for element to be visible then clickable
      @FindBy(xpath = "//a[text()='Login']")
      WebElementFacade loginLink;

      loginLink.withTimeoutOf(Duration.ofSeconds(10)).waitUntilVisible().waitUntilClickable().click();
      // Selenium

      // Wait for element to be clickable with a 10 second timeout
      new WebDriverWait(getDriver(), 60).until(ExpectedConditions.elementToBeClickable(element));

      // Wait for element to be visible with a 10 second timeout
      new WebDriverWait(getDriver(), 60).until(ExpectedConditions.visibilityOf(element));

      // Wait for element to be visible with a 10 second timeout
      @FindBy(xpath = "//h1[text()='Page Header']")
      private List pageHeader;

      new WebDriverWait(getDriver(), 60).until(ExpectedConditions.visibilityOf(element));

Element text contains whitespace Use normalize-space to strip leading and trailing white-space from the text. 

      @FindBy(xpath = "//button[normalize-space()='Continue']")
      private WebElement continue;

Element is not visible to click Use Javascript or Actions to click the element. Javascript is the preferred method. 

      // Javascript - click the element

      evaluateJavascript("arguments[0].click();", element);
      // Actions - move to element and click

      Actions actions = new Actions(driver);
      actions.moveToElement(element).click().perform();

Element is clickable but is covered by an overlay Use Serenity or Selenium to wait for the overlaying element to disappear. Serenity is the preferred method. 

      // Serenity

      // Wait for disappearance of loader then click element
      @FindBy(xpath = "//div[@id='loader']")
      WebElementFacade loaderOverlay;

      @FindBy(xpath = "//a[text()='Yes']")
      WebElement yesLink;

      loaderOverlay.withTimeoutOf(Duration.ofSeconds(10)).waitUntilNotVisible().click();
      yesLink.click();
      // Selenium

      // Wait for element to be visible with a 10 second timeout
      @FindBy(xpath = "//div[@id='loader']")
      WebElement loaderOverlay;

      @FindBy(xpath = "//a[text()='Yes']")
      WebElement yesLink;

      new WebDriverWait(getDriver(), 60).until(ExpectedConditions.invisibilityOfElementLocated(loaderOverlay));
      yesLink.click();

StaleElementReferenceException

Element is temporarily inaccessible due to another action Use try/catch to reattempt to re-access the element. 

      try {
            element.click();
      } catch(StaleElementException e) {
            element.click();
      }

Retry attempt to access the element Use a function to attempt to access the element a number of times. 

      public boolean retryingFindClick(By by) {
      boolean result = false;
      int attempts = 0;

      while(attempts < 2) {
          try {
              element.click();
              result = true;
              break;
          } catch(StaleElementException e) {
          }
          attempts++;
        }
      return result;
      }
